  <action>
    Per D-03, D-04: Create a 7-step onboarding wizard adapted from macOS onboarding.

    **OnboardingStep.kt** — Step interface:
    - Define a common interface/abstract class for wizard steps
    - Each step provides: title (localized), GTK widget content, isComplete check, onComplete callback
    - Steps that are informational (no gate) always report isComplete=true

    **OnboardingWizard.kt** — Wizard container (per specifics: use GtkAssistant):
    - Create a GtkAssistant window with 7 pages
    - Each page corresponds to one step
    - Use GTK_ASSISTANT_PAGE_CONTENT for normal steps, GTK_ASSISTANT_PAGE_CONFIRM for final step
    - On "apply" signal: write `hasCompletedOnboarding = true` to SettingsPersistence, save settings, close wizard
    - Set assistant modal, set reasonable default size (600x400)
    - Use SharedLocalization for all step titles and labels

    **7 Steps (adapted from macOS Pindrop/UI/Onboarding/ files):**

    1. **WelcomeStep** — Welcome message, app description. Pure informational, no gate. Adapted from WelcomeStepView.swift.

    2. **AudioCheckStep** — Per D-04: Soft audio probe (inform, don't block). Detect PipeWire/PulseAudio availability. Show detected audio system. If no audio system found, show warning but allow continuing (softer than macOS which blocks). Adapted from PermissionsStepView.swift.

    3. **ModelSelectionStep** — Let user pick default transcription model. Show available models from KMP model catalog. Adapted from ModelSelectionStepView.swift.

    4. **ModelDownloadStep** — Download the selected model. Show progress placeholder (actual download wiring in Phase 3). Adapted from ModelDownloadStepView.swift.

    5. **HotkeySetupStep** — Per D-04, D-13: Show default hotkey, explain limitations (X11 works, Wayland varies). Allow setting desired shortcut. Show warning badge if runtime can't bind. Offer alternatives: tray click, CLI trigger. Adapted from HotkeySetupStepView.swift.

    6. **AIConfigStep** — Optional AI enhancement setup. Show provider picker from AISettingsCatalog. Allow skipping (core dictation doesn't need AI). Store API key in SecretStorage if provided. Adapted from AIEnhancementStepView.swift.

    7. **ReadyStep** — Confirmation, "You're all set" message. Adapted from ReadyStepView.swift.

    **LinuxCoordinator.kt updates:**
    - In `start()`: if `hasCompletedOnboarding == false`, create and show OnboardingWizard
    - On wizard completion: proceed to normal tray operation
    - Wire `showSettings()` placeholder to be implemented in Task 2

    Agent's discretion: GtkAssistant vs custom wizard, exact widget layouts, GTK signal connection patterns, whether to use GtkBuilder XML or programmatic construction.
