  <action>
    Per D-01, D-02, D-05, D-14: Create the GApplication-based Linux app entry point and coordinator.

    **LinuxApplication.kt** — Main entry point:
    - Define `fun main(args: Array<String>)` as the linuxX64 entry point
    - Create an AdwApplication (libadwaita subclass of GApplication) with app ID "tech.watzon.pindrop"
    - Connect to "activate" signal: create/show the main state, start coordinator
    - Call `g_application_run()` with args
    - Use GTK cinterop wrappers generated from .def files (Plan 01)
    - Per A-05: GApplication lifecycle maps to coordinator pattern — use activate for start, shutdown for cleanup

    **LinuxCoordinator.kt** — Lifecycle coordinator (adapted from macOS AppCoordinator pattern):
    - Own instances of: SettingsPersistence, SecretStorage, AutostartManager, TrayIcon
    - `suspend fun start()`:
      1. Load settings from TOML (via SettingsPersistence.load())
      2. Read `SettingsKeys.hasCompletedOnboarding` — if false, set flag to show onboarding (Plan 03 provides the wizard)
      3. If `SettingsKeys.launchAtLogin` is true, synchronize autostart state via AutostartManager
      4. Initialize TrayIcon
      5. Show tray icon — app is now running
    - `fun quit()`:
      1. Persist any dirty settings (SettingsPersistence.save())
      2. Clean up resources
      3. Call g_application_quit()
    - Expose functions for tray menu actions:
      - `fun showSettings()` — placeholder that Plan 03 will wire to settings dialog
      - `fun toggleAutostart()` — calls AutostartManager and updates setting
      - `fun showAbout()` — shows GTK about dialog
      - `fun quitApp()` — calls quit()
    - The coordinator should use memscoped or stablerefs for C pointer lifecycle management where needed

    **build.gradle.kts changes:**
    - Ensure linuxX64 binary entry point points to `tech.watzon.pindrop.shared.ui.shell.linux.main`
    - Add dependency on `:core` module (for SettingsPersistence, AutostartManager)
    - Add dependency on `:settings-schema` (for SettingsKeys, SettingsDefaults)
    - Add dependency on `:ui-localization` (for SharedLocalization)

    Per A-01: cinterop is proven but large APIs need careful handling. Start with the minimal GApplication subset (new, run, quit, activate signal).

    Per D-08, D-09, D-07: Create the system tray integration and autostart management.

    **TrayIcon.kt** — AppIndicator tray integration:
    - Create AppIndicator instance with ID "pindrop" and category APP_INDICATOR_CATEGORY_APPLICATION_STATUS
    - Set icon name to "pindrop" (expects pindrop.svg in system icon theme or local data dir)
    - Set status to ACTIVE on startup
    - Provide `fun setMenu(menu: GtkMenu)` to attach menu
    - Provide `fun setStatus(active: Boolean)` for state changes
    - Per D-08: Use AppIndicator/KStatusNotifierItem via D-Bus — standard across GNOME (with extension), KDE, XFCE

    **TrayMenu.kt** — GTK menu for tray:
    - Build a GtkMenu with items:
      - "Pindrop" (header/label, disabled)
      - Separator
      - "Settings" → calls LinuxCoordinator.showSettings()
      - "Launch at Login" (check item) → calls LinuxCoordinator.toggleAutostart(), reflects current AutostartManager.isAutostartEnabled()
      - Separator
      - "About Pindrop" → calls LinuxCoordinator.showAbout()
      - "Quit" → calls LinuxCoordinator.quitApp()
    - Use SharedLocalization.getString() for menu labels so they're localized
    - Menu labels to localize: "Settings", "Launch at Login", "About Pindrop", "Quit"

    **TrayFallback.kt** — Graceful degradation for tray-less environments:
    - Per D-09: Fallback for environments without tray support (tiling WMs, minimal setups)
    - Detect if AppIndicator is available (try to create and catch failure)
    - If unavailable, show a small persistent GTK window with a minimal control surface:
      - App icon/logo
      - "Settings" button
      - "Quit" button
    - Log guidance message about tray unavailability
    - Don't crash or silently disappear

    **LinuxCoordinator.kt updates:**
    - In `start()`: initialize TrayIcon, attach TrayMenu
    - Try AppIndicator first; on failure, activate TrayFallback
    - In `toggleAutostart()`:
      - Read current state from AutostartManager
      - Toggle: if enabled → disable, if disabled → enable
      - Update SettingsPersistence with new launchAtLogin value
      - Update menu check item state
    - In `showAbout()`: show a simple GtkAboutDialog with app name, version, copyright

    **pindrop.desktop** — Desktop entry file (installed alongside binary):
    - Standard Desktop Entry format for app registration
    - Include StartupWMClass for proper window grouping (per specifics in CONTEXT.md)
